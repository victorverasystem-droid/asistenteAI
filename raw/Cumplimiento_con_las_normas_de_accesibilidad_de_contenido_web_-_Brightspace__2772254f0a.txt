TITLE: Cumplimiento con las normas de accesibilidad de contenido web - Brightspace
HASH: 2772254f0afa3bda654fc8cc98b1ba58efa002576efb8ebf1b19d89d0097c844
TYPE: web

La mayoría de los consejos que se ofrecen son normas de contenido web establecidas por el Consorcio World Wide Web. Para obtener más información, revise las Pautas de accesibilidad del contenido web.
- Cuando sea posible, utilice los temas HTML para crear el contenido del curso. D2L proporciona plantillas HTML accesibles para agilizar el proceso de creación de contenido. Si debe utilizar otros formatos de archivo (como archivos de video), elija formatos reconocidos por la mayoría de los exploradores y ofrezca el material en varios formatos.
- Utilice un diseño simple que no organice el contenido en tablas o columnas. Cuando la información se organiza de manera simple, los estudiantes pueden leer y comprender más fácilmente. Además, un diseño más simple ayuda a los dispositivos de tecnología de asistencia a interpretarlo y presentarlo con mayor facilidad y los dispositivos móviles y portátiles pueden cambiar su tamaño con menos dificultad.
- Utilice encabezados para comunicar la relación que existe entre las secciones. Utilice el Encabezado 1 para el título, el Encabezado 2 para las secciones principales, el Encabezado 3 para las subsecciones, etc. Si se usan los encabezados correctamente, los usuarios de lectores de pantalla pueden buscar las páginas rápidamente por encabezado y los participantes con discapacidades cognitivas pueden comprender con más facilidad cómo se relacionan las secciones y el contenido.
- Asegúrese de que cada encabezado, elemento y nombre de archivo sea único.
- Incluya una tabla de contenidos que esté vinculada a cada sección. Asegúrese de que existan enlaces de Volver al principio al final de cada sección en los documentos largos.
- Incluya descripciones de texto alternativas (texto alternativo) para todos los gráficos. Utilice comillas dobles (nulo) como texto alternativo si el recurso es un elemento decorativo que no agrega significado al tema. Si el gráfico es un enlace, el texto alternativo debe comenzar con "Vincular a". El editor de Brightspace en Brightspace le indica automáticamente que debe incluir texto alternativo cuando inserte una imagen.
- Incluya leyendas detalladas debajo de tablas y gráficos. Estas leyendas deben explicar qué información transmiten los recursos, incluidas tendencias y estadísticas importantes. Para las tablas, incluya una leyenda con el elemento Leyenda que explica cómo se organiza la tabla. Verifique que las tablas tengan sentido cuando se leen de izquierda a derecha. A menudo, los lectores de pantalla presentan dificultades para transmitir información que se lee de arriba abajo.
- Utilice el mismo texto en la pantalla y en el texto alternativo para los enlaces. Asegúrese de que el texto describa la acción que se producirá cuando se haga clic en el enlace. Nunca utilice texto como "Haga clic aquí" como enlace. Los lectores de pantalla utilizan una lista de enlaces para navegar rápidamente por las acciones de una página No obstante, esto no es posible si los enlaces no son descriptivos.
- Incluya alternativas de texto de contenido multimedia, como archivos de audio o video. Si no cuenta con el tiempo para crear una alternativa de texto completa, incluya una etiqueta descriptiva que resuma el contenido.
- No utilice contenido multimedia intermitente o que titile, ya que puede causarles ataques epilépticos a las personas que sufren de fotosensibilidad. Utilice animaciones que ayuden a transmitir un concepto y que no llamen la atención a recursos estáticos. Utilice una combinación de tamaño, color y prominencia para llamar la atención a los recursos.
- No utilice colores para transmitir significado. Si desea mostrar cómo los conceptos están relacionados entre ellos, utilice una combinación de tamaño, color y etiquetas de texto.
- Asegúrese de que los colores del texto y de los fondos que utiliza en los materiales del curso se contrasten fuertemente.
- Utilice tamaños de fuente pertinentes y asegúrese de que el diseño de la página y el texto se ajuste cuando un usuario cambie el tamaño de fuente. Los usuarios no deben tener que desplazarse horizontalmente.
- Si utiliza archivos PDF para el contenido, escanéelos con reconocimiento óptico de caracteres (OCR) para que los lectores de pantalla puedan leer el texto. Escanee las páginas con varias columnas de una columna a la vez para que el reconocimiento óptico de caracteres funcione como se espera. Agregue marcadores a las secciones principales para que sea más fácil navegar el contenido.
- Si crea documentos PDF a partir de Microsoft Word u otro procesador de textos, asegúrese de formatear los títulos y las secciones con encabezados para que puedan etiquetarse correctamente en el PDF.
- Si los usuarios utilizan un software de reconocimiento de voz para leer el texto en voz alta o resaltarlo a medida que leen, pruebe el software para asegurarse de que funcione correctamente con las herramientas y las configuraciones específicas que desea crear. Por ejemplo, si deshabilita las funciones del botón derecho durante los cuestionarios, esto puede evitar que algunos software de reconocimiento de voz realicen el seguimiento del texto.
